# FORGE v223 Changelog — 2026-03-17

## Theme: Coach Polish + DNN Accuracy + Context Expansion

---

### Coach Intercept Card — Markdown Rendering Fixed (coach-triggers.js)
- Added `_renderCoachText(el, text)` helper: HTML-escapes first (XSS safe), then converts `**text**` → `<strong>`, `*text*` → `<em>`, `\n` → `<br>`
- All coach message surfaces now use `_renderCoachText`: intercept card, daily brief (cached restore + live update + new card), debrief streaming preview
- Lesson: LLM ignores "plain text" instructions in practice — always strip markdown client-side

### _fireCoachMessage SSE — Buffer Fix (coach-triggers.js)
- Was: `decoder.decode(value)` + `chunk.split('\n')` (no carry-over buffer → drops tokens at chunk boundaries)
- Now: canonical pattern `buf += decoder.decode(value, {stream:true}); const lines = buf.split('\n'); buf = lines.pop()`

### DNN Steps Counter — Uses User's Custom Goal (dashboard-history.js)
- `stepsDone` now checks `_todayStepsData.goal` (from `getTodaySteps()`) instead of hardcoded `8000`
- Steps pill label is now dynamic: `"10k+ steps"` or whatever the user set via `setStepsGoal()`
- Tap hint progress shows real goal: `"2.5K / 10K steps"` instead of always `"/ 8K"`

### buildUserContext Expansion (rag-search.js)
Added 3 new context blocks the coach can now reference:
1. **Today's nutrition** — actual calories + protein from `forge_meals[todayKey]` (not just 7-day avg)
2. **Today's readiness** — score, sleep hours, HRV, resting HR from `forge_readiness[todayKey]`
3. **Body weight trend** — direction + delta across last 4 `forge_bodyweight` entries (gaining/losing/stable)

### Coach Ask-Answer — Markdown Render + FX (index.html)
- `_askCoach`: added `"Plain text only, no markdown."` to system prompt
- Post-stream: converts markdown to safe HTML via regex (same pattern as `_renderCoachText`)
- `sndTap()` fires on first token arrival; `addRipple()` on bubble when stream completes

### DNN Week Plan — Uses User's forge_split (index.html)
- `renderCoachPlan()` reads `forge_split` localStorage; when ≥3 non-Rest days configured, uses the actual split instead of hardcoded goal-based SPLITS template
- Falls back to goal template when split is not configured

### Version
- `js/config.js`: `FORGE_VERSION = 'v223'`
- `sw.js`: cache bumped to `forge-v223`

## Bug Fix Summary
| Bug | Root Cause | Fix |
|---|---|---|
| Coach intercept shows `**bold**` | LLM ignores plain-text instruction | `_renderCoachText` helper on all coach surfaces |
| SSE drops tokens | Unbuffered decode with no carry-over | Canonical buffered SSE pattern |
| DNN steps shows "0 / 8K" after logging | Hardcoded 8000 threshold + label | Read from `getTodaySteps().goal` |
| Coach unaware of today's data | buildUserContext only had history | Added today's meals, readiness, weight trend |
